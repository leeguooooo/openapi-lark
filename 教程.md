# openapi-lark 用法

你只需要装上 skill，然后跟 Claude / agent 说人话。agent 会自己跑命令、改配置、处理报错。

## 装一次

```bash
npx skills add leeguooooo/openapi-lark
```

## 直接跟 agent 说

### 第一次接入项目

> 帮我把这个项目的接口同步到飞书：<贴飞书 wiki URL>

agent 会：
1. 在你项目里建 `.openapi-lark.yaml`
2. 推全部接口到飞书 wiki（每个接口一个子文档）
3. 报进度

### 改了 openapi 之后

> 重新同步一下接口文档

agent 会：
- 自动跳过没变的接口（hash 缓存）
- 只重推改了的

### 想看效果

> 同步前先让我看下渲染效果

agent 会跑 `--dry-run` 生成本地 md 给你看。

### 出问题了

> 同步报错了

直接把报错截图 / 粘给 agent，它会按内置故障诊断流程修。常见的：
- 父节点标题被改 → agent 加 `parentTitle` 配置
- `missing scope` → agent 让你跑一次 `lark-cli auth login --scope ...`
- 响应字段不全 → agent 删 lockfile 重推

## 第一次需要你手工做的事

只有一件：**在飞书 wiki 手工建一个空的父节点**（agent 没法替你点这个按钮）。复制它的 URL 给 agent 就行。

> 提示：不要复用已有内容的节点，sync 会清空它。

## 如果你不用 agent

要纯手动跑，看 [SKILL.md](./SKILL.md) 的「新项目快速开始」节，里面有完整命令。
